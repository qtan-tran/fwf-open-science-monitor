## 🏛️ Origins & Authorship
This repository was created and is maintained by **Quoc-Tan Tran**, Open Science Researcher at the **Faculty of Sociology, Bielefeld University**, with the technical assistance of **Claude AI**. 

It started as a small attempt to make sense of funding data in a way that others could also follow and reuse. The idea is not to build a perfect system, but to show how simple automated pipelines and basic web tools can already make research funding and outputs a bit more transparent and easier to explore.

## ⚡ The Vision
The goal here is quite straightforward: take messy, raw funding data and turn it into something you can actually look at and understand. By showing things like open access rates, publication trends, and institutional activity, this monitor tries to give a clearer picture of what’s going on—hopefully in a way that supports more open and reproducible science.

## ✨ Key Features

- **📊 Six-Axis Dashboard** — Headline metrics (OA rates, funding efficiency) with interactive Recharts visualizations.
- **🔍 Deep-Dive Explorer** — 10 unique analysis modes including researcher search, publication breakdowns, and funding scatter plots.
- **⚡ High-Performance ETL** — Python 3.12 pipeline that daily syncs, cleans, and computes metrics from the FWF Open Research API.
- **🚀 Production Ready** — In-memory LRU caching (5-min TTL), full type-safety with Prisma, and system-aware Dark Mode.
- **📦 One-Command Setup** — Fully containerized environment for instant local development and testing.

## 🚀 Quick Start (Localhost)

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running
- [Node.js](https://nodejs.org/) v20+
- [Python 3](https://www.python.org/) (only needed for the ETL pipeline)
- [Git](https://git-scm.com/)

### 1. Clone & Configure

```bash
git clone https://github.com/qtan-tran/fwf-open-science-monitor.git
cd fwf-open-science-monitor
```

Copy the example environment file:

- **macOS / Linux:** `cp .env.example .env`
- **Windows (PowerShell):** `Copy-Item .env.example .env`

Edit `.env` and add your FWF API key (get it from https://openapi.fwf.ac.at/fwfkey):

```
FWF_API_KEY=your_key_here
```

> **Note:** The API key is only required to run the ETL pipeline. You can browse the dashboard without one (it will show empty charts until data is loaded).

### 2. Start the Database

```bash
docker compose up db -d
```

Wait a few seconds for PostgreSQL to become ready.

### 3. Install Dependencies & Set Up the Schema

```bash
cd apps/web
npm install
npx prisma generate
npx prisma db push
cd ../..
```

### 4. Start the App

```bash
cd apps/web
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

> **💡 Tip:** The database starts empty. To see real data, run the ETL pipeline (see [ETL Guide](#-etl-guide) below) to sync live data from the FWF API, or load a pre-built `seed/seed.sql` if one is available (see [Loading Seed Data](#loading-seed-data) below).

## Loading Seed Data

A `seed/seed.sql` file is not included in the repository — it is generated by running the ETL pipeline at least once. After the ETL has run, you can export a snapshot for faster future loads:

```bash
# Export a seed snapshot (macOS/Linux):
docker compose exec db pg_dump -U postgres fwf_monitor > seed/seed.sql

# Export a seed snapshot (Windows PowerShell):
docker exec (docker compose ps -q db) pg_dump -U postgres fwf_monitor | Out-File -Encoding utf8 seed/seed.sql
```

Once you have a `seed/seed.sql`, load it with:

**macOS / Linux (convenience script — handles both paths automatically):**
```bash
./seed/load_seed.sh
```

**macOS / Linux (manual, with psql installed locally):**
```bash
psql "$DATABASE_URL" -f seed/seed.sql
```

**macOS / Linux (via Docker, no local psql needed):**
```bash
docker compose exec -T db psql -U postgres -d fwf_monitor < seed/seed.sql
```

**Windows (PowerShell, via Docker):**
```powershell
Get-Content seed/seed.sql | docker exec -i (docker compose ps -q db) psql -U postgres -d fwf_monitor
```

**Or use any database GUI** (pgAdmin, DBeaver, DataGrip): connect to `localhost:5432`, user `postgres`, password `postgres`, database `fwf_monitor`, and run the contents of `seed/seed.sql`.

## 🔄 ETL Guide

The ETL pipeline fetches data from the FWF Open Research API, cleans it, and populates the PostgreSQL database.

### Prerequisites

1. **Get an API Key:** Grab a free key at https://openapi.fwf.ac.at/fwfkey
2. **Configure:** Open your `.env` and set `FWF_API_KEY=your_key_here`
3. **Database running & schema applied** (see Quick Start steps 2–3 above)

### Option A: Run via Docker (no Python setup needed)

```bash
docker compose run --rm etl
```

### Option B: Run natively (faster iteration, no Docker build required)

**macOS / Linux:**
```bash
cd etl
pip install -r requirements.txt
python -m src.pipeline
```

**Windows (PowerShell):**
```powershell
cd etl
pip install -r requirements.txt   # use --user if global install is blocked
python -m src.pipeline
```

> **Tip:** After the ETL completes, export a `seed/seed.sql` snapshot for faster future loads (see [Loading Seed Data](#loading-seed-data) above).
